# Decision Policy v1.0

## Recording Criteria
Record a Memory Event when ANY are true:

1. **Architectural Impact**: Affects system architecture
2. **Dependency Introduction**: Adds new dependencies  
3. **Significant Trade-off**: Resolves non-trivial trade-offs
4. **Precedent Setting**: Establishes patterns for future

## Quality Standards
All Memory Events must include:
- **Rationale**: Minimum 20 characters
- **Alternatives**: At least 1 considered option  
- **Impact**: low/medium/high classification
- **Decision Framework**: Principles used
